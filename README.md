## Architecture

```
nodetalk/
├── backend/
│   ├── cmd/server/        → Go server entry point
│   ├── internal/
│   │   ├── config/        → TOML config (Priority source)
│   │   ├── crypto/        → Argon2id hashing, AES-256-GCM
│   │   ├── db/            → BadgerDB (Encrypted at rest)
│   │   ├── store/         → Repository layer
│   │   ├── auth/          → Session & Token management
│   │   ├── api/           → REST Handlers
│   │   ├── ws/            → WebSocket Hub
│   │   ├── voice/         → UDP Voice Router
│   │   └── models/        → Data structures
│   └── tests/             → Comprehensive Stress & Unit tests
├── client-web/            → React + Vite + TypeScript
└── client-desktop/        → Wails + Go + React
```

### Security Architecture (Server-Trusted)

NodeTalk uses a **Server-Trusted Encryption** model. While all communication is encrypted using industry-standard AES-256-GCM, the server acts as the central authority for key management and distribution.

1.  **Master Password**: A server-side master password derives the **Key Encryption Key (KEK)** using Argon2id.
2.  **Data at Rest**: The database (BadgerDB) is encrypted at rest using a **Data Encryption Key (DEK)** which is wrapped by the KEK.
3.  **Channel Keys**: Every channel has a unique AES-256 key. These keys are generated by the server, stored encrypted (wrapped by KEK), and distributed to authorized members via secure WebSocket channels.
4.  **Client Encryption**: Messages are encrypted in the client before transmission. The server routes the opaque ciphertext. 

> [!NOTE]
> Because the server manages and distributes the channel keys, it is within the trust boundary. This ensures seamless multi-device support and easier recovery compared to strict E2EE.

## Quick Start

### 1 — Docker (Recommended)
The fastest way to deploy NodeTalk is using Docker.

*   **Development**: Use the root `docker-compose.yml` to build from source.
*   **Production**: Use our pre-built images. See the [Production Example](./examples/docker-compose/production/) for a ready-to-use setup.

```bash
# To run the development version:
docker-compose up -d
```

See [DOCKER.md](./DOCKER.md) for detailed configuration and the [Examples](./examples/) directory for more deployment scenarios.

### 2 — Native Build (Go & Node)
```bash
# Backend
cd backend && go run ./cmd/server

# Web Client
cd client-web && npm install && npm run dev
```

### 3 — Desktop App (Wails)
```bash
make wails/build/mac  # or wails build for your OS
```
See [desktop_build_guide.md](./desktop_build_guide.md) for more info.

## Configuration

Settings are managed via `backend/config.toml`. 

| Section | Key | Description |
|---|---|---|
| `server` | `http_port` | API & WebSocket port (default 8080) |
| `server` | `udp_port` | Voice Chat port (default 9090) |
| `security` | `master_password` | Master key for all encryption |
| `database` | `path` | Path to BadgerDB storage |

We include a high-concurrency stress testing suite to ensure stability.

```bash
cd backend

```
